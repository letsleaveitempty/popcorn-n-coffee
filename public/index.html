Synchronizing git repositories - how to?



Step 1: Git Remotes


What is a git remote repository? Remote repositories are versions of your project that are hosted on the Internet(e.g. GitHub) or network somewhere (e.g. on your computer).

The goal is to synchronize your local, origin and upstream remote.

  • UPSTREAM - upstream project repository on GitHub (original one)
  • ORIGIN - forked project repository on your personal GitHub account
  • LOCAL - cloned repository on your computer

In your application directory:

git remote -v

# shows current remotes

git remote add upstream htttps://github.com/otherusername/otherrepositoryname.git

# creates upstream remote

git remote -v

# please, verify if new remote is created



Step 2: Sync Upstream and Local Remote


git fetch upstream

# grabbing upstream branches

git branch -va

# see branches list from all remotes

If there were branches added to original repository (upstream), they are automatically added to local remote (and visible on the list via git branch -va). In case there were changes on a branch you had on your local remote before, you have to merge each branch separately:

git checkout master

# make sure you are on branch master

git merge upstream/master

# merge upstream master into your local master

Repeat with all changed branches (if you want to have them synchronized :) )

git checkout nameofthebranch

git merge upstream/nameofthebranch


Tadaam! From now on you can use actual version of original repository (on your computer).



Step 3: Sync Local and Origin Remote


To synchronize your origin remote, you have to push changes from your computer:

git checkout nameofthebranch

# make sure you are on the branch you need to sync

git push origin nameofthebranch

# pushes branch into origin, creates a new one in case it doesnâ€™t exist

Repeat for all branches you want to synchronize. Be careful with names: git push nameofthebranch tries to push version from your local remote into original one (upstream). And it will fail...


Tadaam, taadaaam, taaadaaaam!!! All remotes are the synchronized, plus you have your commits saved on origin remote.
